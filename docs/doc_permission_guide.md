# 飞书文档权限设置指南

## 问题
通过 API 创建的文档，默认只有应用本身有权限，其他用户无法打开。

## 解决方案

### ✅ 方案1（推荐）：发送链接，手动设置权限
最简单直接的方法。

**使用脚本**: `examples/send_doc_link_to_you.py`

**步骤**:
1. 运行脚本创建文档
2. 脚本会把文档链接通过飞书消息发给你
3. 在飞书中打开文档
4. 点击右上角的「分享」按钮
5. 添加需要的协作者或设置文档公开
6. 完成！

### ✅ 方案2：通过飞书开放平台给应用设置权限
在飞书开放平台：
1. 进入应用 → 权限管理
2. 添加权限: `docx:document:edit`
3. 发布新版本
4. 然后可以使用代码添加协作者

### ✅ 方案3：把文档放到文件夹中（推荐用于项目）
1. 在飞书中创建一个文件夹
2. 把文件夹设置为团队可见
3. 以后创建文档时，通过 API 指定文件夹路径
4. 文档会继承文件夹的权限

## 使用示例

### 快速创建文档并获取链接
```bash
python examples/send_doc_link_to_you.py
```

```
文档链接会自动发送到你的飞书中
```

## 需要的权限

| 权限 | 说明 | 状态 |
|------|------|------|
| `docx:document:create` | 创建文档 | ✅ 已配置 |
| `docx:document:edit` | 编辑文档/添加协作者 | 可选 |

## 最佳实践

1. **项目初期**: 用方案1（发送链接手动设置），简单可靠
2. **稳定运行后**: 用方案2（API 设置权限），自动化程度高
3. **团队使用**: 用方案3（文件夹权限），方便管理

## 常见问题

### Q: 为什么打开文档提示没有权限？
A: 这是正常的。通过 API 创建的文档，默认只有应用有权限。你需要通过飞书界面手动添加权限。

### Q: 可以通过 API 自动设置权限吗？
A: 可以，但需要先配置 `docx:document:edit` 权限。不过手动设置更简单，推荐先用方案1。

### Q: 如何让整个团队都能访问？
A: 创建一个团队文件夹，让文档都在文件夹中创建，文档会继承文件夹的权限。
